# Nextcloud

All-in-one Nextcloud deployment with Nginx Proxy Manager (NPM) for reverse proxying.

## Usage

```bash
docker compose up -d
```

## Services

| Service | Port | Purpose |
|---|---|---|
| Nginx Proxy Manager | 81 | Reverse proxy admin UI |
| Nextcloud AIO | 8085 | Nextcloud master container |

## Volumes

| Host path | Container path | Purpose |
|---|---|---|
| `./npm/data` | `/data` | NPM data and config |
| `./npm/letsencrypt` | `/etc/letsencrypt` | SSL certificates |
| `nextcloud_aio_mastercontainer` | `/mnt/docker-aio-config` | AIO config (Docker volume) |
| `/var/run/docker.sock` | `/var/run/docker.sock` | Docker socket (for container mgmt) |
| `/srv/nextcloud-data` | `/mnt/ncdata` | Nextcloud file storage |

## Permissions

```bash
sudo chown -R 1000:1000 apps/nextcloud/npm
sudo chown -R 1000:1000 /srv/nextcloud-data
```

Nginx Proxy Manager data and the Nextcloud data directory must be writable by the container's user (uid 1000). The AIO master container uses a Docker volume and manages its own permissions.

## Backup

```bash
# NPM config
tar czf backup-nextcloud-npm.tar.gz apps/nextcloud/npm/data

# Nextcloud files
sudo tar czf backup-nextcloud-data.tar.gz /srv/nextcloud-data

# AIO master container config (Docker volume)
docker run --rm -v nextcloud_aio_mastercontainer:/config alpine tar czf - -C /mnt/docker-aio-config . > backup-nextcloud-aio-config.tar.gz
```

Do **not** back up `npm/letsencrypt` — SSL certificates are auto-renewed by Let's Encrypt.

## Restore

1. Extract the NPM data backup into `apps/nextcloud/npm/data`.
2. Extract the Nextcloud data backup into `/srv/nextcloud-data`.
3. Restore the AIO Docker volume:
   ```bash
   docker run --rm -v nextcloud_aio_mastercontainer:/config alpine sh -c 'tar xzf - -C /mnt/docker-aio-config' < backup-nextcloud-aio-config.tar.gz
   ```
4. Fix ownership: `sudo chown -R 1000:1000 apps/nextcloud/npm /srv/nextcloud-data`
5. Start the containers: `docker compose up -d`

## Notes

- The `APACHE_PORT=11000` in `compose.yaml` must match the backend port in your NPM proxy config.
- NPM proxy host configs in `npm/data/nginx/proxy_host/` contain sanitized example domain names — replace with your own domains.
- SSL certificates in `npm/letsencrypt/` are auto-generated by Let's Encrypt and must not be committed.
- The `update-nc-dns.sh` script is a template — fill in your Cloudflare API credentials before use.
